01 — Was ist eine IP-Adresse?

Eine IP-Adresse ist wie eine Hausnummer im Internet — sie identifiziert eindeutig ein Gerät im Netzwerk.

03 — IP-Klassen

IPv4-Adressen sind in Klassen eingeteilt. Die Klasse bestimmt, wie groß das Netz ist.

🏷️ IP-Klassen Übersicht

B 10xx xxxx 128.0.0.0 – 191.255.255.255 16.384 65.534
C 110x xxxx 192.0.0.0 – 223.255.255.255 2.097.152 254
✅
Merkrege: Klasse C = 192–223 im ersten Byte. Das ist die häufigste Klasse in Schulaufgaben (z.B. 192.168.x.x).

04 — Was ist Subnetting?

Subnetting bedeutet: ein großes Netz in kleinere Teilnetze aufteilen — wie Wände in einem großen Haus einziehen.

Stell dir vor: Du hast ein Bürogebäude (= ein /24 Netz mit 254 Adressen). Du willst für jede Abteilung ein eigenes Netz — damit der Verkehr getrennt bleibt und du weißt, welches Gerät wo ist.

🏢 Ein /24 Netz wird aufgeteilt

→ Aus einem /24 Netz entstehen durch Subnetting z.B. 16 kleinere /28 Netze.

Warum Subnetting?

1

Ordnung & Struktur

Jede Abteilung/Raum bekommt ein eigenes Subnetz — klar getrennt und übersichtlich.

2

Sicherheit

Subnetze können voneinander isoliert werden — z.B. darf das Büronetz nicht das Servernetz erreichen.

3

Weniger Broadcast-Traffic

Broadcasts bleiben im Subnetz — das entlastet das gesamte Netz.

06 — Bits borgen

Subnetting funktioniert so: Man nimmt Bits vom Host-Teil und gibt sie dem Netz-Teil. Das erzeugt mehr, aber kleinere Subnetze.

S = Subnetz-Bit (geborgt)
H = Host-Bit (bleibt)
4 Subnetz-Bits → 2⁴ = 16 Subnetze
4 Host-Bits → 2⁴−2 = 14 nutzbare Hosts je Subnetz
⚠️
Mehr geborgene Bits = mehr Subnetze, aber weniger Hosts pro Subnetz. Das ist immer ein Kompromiss!

Die Formel

Subnetze = 2^(geborgene Bits)
Hosts pro Subnetz = 2^(verbleibende Bits) − 2 (−Netz −Broadcast)

08 — Subnetz-Tabelle erstellen

Wenn du die Blockgröße weißt, kannst du alle Subnetze aufzählen. Blockgröße = Gesamtadressen pro Subnetz.

Beispiel: /28 → Blockgröße = 16. Subnetze beginnen bei 0, 16, 32, 48, ...

📋 Subnetz-Tabelle für 192.168.50.0/28
Nr. Subnetz-ID Erster Host Letzter Host Broadcast
1.0.1.14.15
2.16.17.30.31
3.32.33.46.47
4.48.49.62.63
...............
16.240.241.254.255

Merkhilfe für jedes Subnetz

①

Subnetz-ID

= Subnetz-Nummer × Blockgröße  (0×16=0, 1×16=16, 2×16=32 ...)

②

Erster Host

= Subnetz-ID + 1

③

Broadcast

= Subnetz-ID + Blockgröße − 1  (0+16−1=15)

④

Letzter Host

= Broadcast − 1
